# Установка, запуск и авто-режим

## Установка приложения

1. Скачайте `VK-Turn-Proxy-Agent.dmg` из [Releases](../../releases)
   (универсальный: Apple Silicon + Intel).
2. Откройте DMG, перетащите **VK Turn Proxy Agent** в **Applications**, запустите.
3. Первый запуск (сборка без подписи): правый клик → **Open** → **Open**. Либо:
   `xattr -dr com.apple.quarantine "/Applications/VK Turn Proxy Agent.app"`.
4. В трее → **Edit config…**, заполните настройки ([config.md](config.md)).

## Где лежит конфиг

Один файл: `~/Library/Application Support/VKTurnProxy/config.json`. Его
используют приложение, сервис и CLI. Открыть/показать — кнопки **Edit config…**
и **Reveal** в трее. Импорт из iOS:

```shell
vk-turn-socks -import 'vkturnproxy://import?data=…'
```

## Использование БЕЗ Surge

Приложение поднимает локальный SOCKS5-прокси на `127.0.0.1:1080` (и HTTP на
`127.0.0.1:1087`). Укажите его в системных настройках сети или в конкретном
приложении:

- **Системный прокси:** Системные настройки → Сеть → ваше подключение →
  Details → Proxies → включите **SOCKS Proxy**, хост `127.0.0.1`, порт `1080`.
- **Отдельное приложение** (браузер и т.п.): укажите SOCKS5 `127.0.0.1:1080` в
  его настройках прокси.

Управление — из трея: **Connect / Disconnect** (когда выключен Auto).

## Использование С Surge

В `[Proxy]` добавьте прокси, в правилах — направьте нужный трафик на него:

```ini
[Proxy]
VKTurn = socks5, 127.0.0.1, 1080, udp-relay=true

[Rule]
# трафик самого сервиса — напрямую, иначе он зациклится (см. Enhanced Mode ниже)
PROCESS-NAME,vk-turn-socks,DIRECT
FINAL,VKTurn
```

## Авто-failover (рекомендуется)

Цель: пока прямой интернет работает — идти напрямую; как только он пропадает —
автоматически подключиться через прокси; когда возвращается — снова напрямую.
Работают две независимые части.

**1. Маршрут переключает Surge** — группой `fallback`, которая сама
health-check'ает DIRECT и уходит на VKTurn, если DIRECT недоступен:

```ini
[Proxy]
VKTurn = socks5, 127.0.0.1, 1080, udp-relay=true

[Proxy Group]
Auto = fallback, DIRECT, VKTurn, url=http://www.gstatic.com/generate_204, interval=30, timeout=3

[Rule]
# проба связности — всегда напрямую (в обход туннеля)
DOMAIN,www.gstatic.com,DIRECT
# трафик самого сервиса — напрямую (без петли)
PROCESS-NAME,vk-turn-socks,DIRECT
IP-CIDR,127.0.0.1/32,DIRECT
FINAL,Auto
```

**2. Процесс запускает агент** — в трее включите **Auto (failover)** и
**Launch at login**. Агент проверяет прямой доступ и **поднимает** движок
только когда прямой интернет пропал, **гасит** — когда вернулся. Так туннель к
VK держится лишь когда реально нужен.

Отдельно в Surge настраивать больше ничего не нужно — только блок выше. Если
Surge не используете, части 2 достаточно для управления процессом, а трафик
направляйте на прокси как в разделе «Без Surge».

Что происходит само: агент стартует при входе → при потере прямого доступа
поднимает прокси и Surge уводит трафик в туннель → при восстановлении гасит
прокси и Surge возвращает DIRECT.

## Surge Enhanced Mode (системный TUN) — как избежать петли

В Enhanced Mode Surge заворачивает весь трафик в системный TUN, включая
исходящие сокеты самого сервиса к VK-релею. Если их проксировать, получается
петля (сервис → Surge → сервис) и подключения нет. Два способа исправить:

- **Правило в Surge (простое, рекомендуется):** отправьте трафик сервиса
  напрямую — оно уже есть в блоках выше:
  ```ini
  PROCESS-NAME,vk-turn-socks,DIRECT
  ```
  Поставьте его выше `FINAL`. Этого достаточно, чтобы петли не было.

- **Bind к интерфейсу (без Surge-правил):** в конфиге задайте физический
  интерфейс — движок будет ходить в обход TUN на уровне сокетов:
  ```json
  "bind_interface": "en0"
  ```
  `en0` — обычно Wi-Fi, `en1` — Ethernet (посмотреть: `ifconfig` / «Системные
  настройки → Сеть»). Пусто = обычная маршрутизация.

## Уведомления

Агент показывает системные уведомления:

- **Подключено** — когда реально установилось соединение (в ручном и авто-режиме).
- **Прямой доступ пропал — подключаю прокси** / **восстановлен — отключаю** —
  когда авто-режим обнаружил изменение доступности прямого интернета.
- **Отключено** — когда авто-режим успешно отключил прокси.

При первом запуске macOS спросит разрешение на уведомления — разрешите.

## Другие способы запуска

**Терминал:**

```shell
vk-turn-socks          # берёт конфиг из app-support
```

**Фоновый сервис (launchd), без приложения:**

```shell
scripts/service.sh install     # поставить и запустить (автозапуск при входе)
scripts/service.sh start|stop|status|logs|uninstall
```

## Логи

В трее — кнопка **Logs**: окно с логом текущей сессии (обновление в реальном
времени, Reveal/Clear). Если Connect не подключается — причина будет там.
Файл: `~/Library/Application Support/VKTurnProxy/agent.log`.

## Решение капчи

Обычно решается автоматически. Если нет — в трее появится кнопка
**Solve captcha…** (откроется окно, пройдите проверку). Альтернатива — вход по
`cookie_header` в конфиге (капча тогда не нужна).
